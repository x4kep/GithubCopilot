## ğŸ§© Architecture

### Web Components

The application uses native Web Components (Custom Elements) for modular, reusable UI:

#### 1. **MonthInputRow** (`<month-input-row>`)

**File:** `components/month-input-row.js`

**Purpose:** Renders a single month's income and expense input fields.

**Attributes:**

- `month-index` (0-11): Index representing the month

**Key Features:**

- Auto-validation on input/blur events
- Bootstrap-styled input groups
- Accessible ARIA labels
- Data attributes for querying

**Lifecycle:**

- `constructor()`: Initializes month index and name
- `connectedCallback()`: Renders HTML and attaches event listeners
- `render()`: Generates input fields with proper structure
- `attachEventListeners()`: Sets up validation handlers

---

#### 2. **MonthInputList** (`<month-input-list>`)

**File:** `components/month-input-list.js`

**Purpose:** Container that renders all 12 month input rows.

**Key Features:**

- Generates 12 `<month-input-row>` components dynamically
- No internal state management

**Lifecycle:**

- `connectedCallback()`: Renders all month rows

---

#### 3. **IncomeExpenseChart** (`<income-expense-chart>`)

**File:** `components/income-expense-chart.js`

**Purpose:** Renders an interactive bar chart comparing income vs expenses.

**Key Features:**

- Chart.js integration
- Dynamic data fetching from DataService
- Chart destruction/recreation on tab switch
- PNG export functionality
- Responsive chart sizing

**Lifecycle:**

- `constructor()`: Initializes chart reference
- `connectedCallback()`: Renders canvas and sets up listeners
- `initializeChartListener()`: Listens for tab switches and download button
- `updateChart()`: Fetches data and recreates chart
- `downloadChart()`: Exports chart as PNG with date stamp
- `disconnectedCallback()`: Cleans up chart instance

**Chart Configuration:**

- Type: Bar chart
- Datasets: Income (green #4CAF50) & Expenses (light green #81C784)
- Responsive: Yes
- Title: "Monthly Income vs Expenses"
